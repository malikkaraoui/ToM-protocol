## Analyse détaillée des erreurs

### Erreur observée
```
[20:48:13] Error: send failed after 3 attempts: 
connection to 4da86cb4... failed: timed out
```

### Timeline
- `20:45:00` : Peer connected ✅
- `20:45:10` : Message "cc" received ✅
- `20:46:42` : First send failure (1m32s après connexion)
- `20:48:13` : Connection timeout (3m13s après connexion)

### Cause probable
**Keepalive mort** ou **idle timeout** trop court.

La connexion meurt silencieusement après ~1-2 minutes d'inactivité.
Le send détecte que la connexion est morte → timeout après 3 retry.

### Hypothèse buffer vs timeout
- ❌ Pas juste buffer overflow
- ✅ **Connexion QUIC morte** (idle timeout)
- ⚠️ **Keepalive non fonctionnel**

### Fix suggéré
1. Vérifier QUIC idle_timeout config
2. Implémenter keepalive robuste (ping/pong)
3. Détecter connection loss plus tôt
4. Reconnexion automatique
